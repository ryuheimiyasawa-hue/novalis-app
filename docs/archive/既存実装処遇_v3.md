## 5. NEEDS-DECISION（2件、判断待ち）

### Q1. `src/app/api/chat/stream/route.ts`（チャットストリーミング）の扱い
- 既存実装あり（9286 bytes）。Claude の SSE ストリーミング前提
- **問い**: MVP でストリーミング体験を維持するか、シンプルに `chat/send` の同期レスポンスに統一するか
- 判断材料:
  - GPT-4o-mini はストリーミング可能なので技術的には継続可
  - 一人開発なので**同期に統一して実装量を削る**選択肢あり
  - Bot 経由の応答はストリーミング不要（Messenger は完成形メッセージを送る）
- **推奨**: MVP は **同期統一**、Phase 2 でストリーミング復活

### Q2. 月次チャットカウンタのリセット方式
- **暦月の1日0時にリセット**で確定済みだが、実装方式が未定
- 選択肢:
  - (a) **Lazy reset**: ユーザー送信時に「最終リセット日時 < 今月1日」なら 0 に戻す（cron 不要、シンプル）
  - (b) **cron**: Supabase scheduled function で毎月1日0時に一斉 UPDATE
- **推奨**: **Lazy reset**（一人開発前提でインフラ不要、確実）。タイムゾーンは **JST**
