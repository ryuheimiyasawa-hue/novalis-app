## 1. プロジェクト概要

### 1-1. プロダクト

- **プロダクト名**: Philippine Community（仮称）
- **一言定義**: 日本在住フィリピン人（永住者・配偶者・定住者・特定技能を中心とした3万人規模）の生活上の困りごとを、AIによる一次情報提供と提携士業へのスムーズなエスカレーションでワンストップ解決するプラットフォーム

### 1-2. 運営体制

| 主体 | 役割 |
|---|---|
| 株式会社ノヴァリス | 技術提供・プラットフォーム運営 |
| フィリピン人ネットワーク（協業先） | コミュニティリーチ・運営パートナー |
| 提携士業（弁護士・行政書士・社労士） | エスカレ受け・監修。必要に応じ士業ネットワーク経由で税理士等を追加可能 |

### 1-3. ターゲット

| 層 | 想定特性 | MVP扱い |
|---|---|---|
| 永住者 | 30-50代、N3-N1、月収中位、家族・教育・住居に関心 | **メイン** |
| 日本人配偶者 | 配偶者ビザ更新、子育て、夫婦間トラブル | **メイン** |
| 定住者 | 永住者と類似、行政手続き・社会保障 | **メイン** |
| 特定技能 | 20-30代、N4-N3、転職・VISA更新・社会保険 | **メイン** |
| 留学生・家族滞在・技人国等 | — | サブ（拒まないが主軸ではない） |
| 技能実習 | — | **対象外**（監理団体トラブル回避のため意図的にスコープ外） |

- 規模目標: MVP **3万人規模**（30万人は長期ビジョン）

## 2. MVP 機能要件

### 2-1. 機能一覧と要件マトリクス

| # | 機能 | 担当チャネル | AI回答範囲 | エスカレ先 | MVP |
|---|---|---|---|---|---|
| 1 | VISA更新サポート | Bot+Web | 在留資格別の制度説明、必要書類リスト、期限、申請場所 | 行政書士 | ✅ |
| 2 | 社会保険 | Bot+Web | 健保・年金・介護保険の制度説明、加入義務、保険料の仕組み | 公的窓口（年金事務所・区役所）／個別質問は社労士へ | ✅ |
| 3 | 夫婦間トラブル | Web中心（Botは入口のみ） | DV相談窓口、離婚制度の一般説明、親権の枠組み、ホットライン番号 | 弁護士／緊急時はDV相談ナビ等公的窓口を最優先 | ✅ |
| 4 | 学校関連 | Bot+Web | 就学手続き、PTA・給食・学費補助、日本の学校文化 | 弁護士 or 自治体相談員 | ✅ |
| 5 | 行政手続き | Bot+Web | 転入届、住民票、印鑑登録、各種証明書、国保・国民年金 | 行政書士 | ✅ |
| 6 | 士業エスカレフォーム | Web専用 | （回答機能ではなくフォーム） | 1-5から流入→該当士業に通知 | ✅ |
| 7 | フィリピン人飲食店紹介 | Bot+Web | 運営選定型カタログ（ユーザー投稿なし） | リスクなし | ✅ |

### 2-2. AI 回答の安全方針（Conservative Whitelist）

- AIは「**ゼロリスクの一般情報**」のみ回答する
- 個別性・具体性が含まれる質問は**全件士業エスカレフォームに誘導**
- 個別性検知の二段階構成:
  1. **キーワード検知**: 「私の場合」「具体的に」「いくら」「私のビザ」「請求できますか」「不当な」等の個別性キーワード
  2. **LLM自己判定**: 「これは個別事案性が含まれるか」を毎回判定
- システムプロンプト命令: **迷ったらエスカレに倒せ**（偽陰性ゼロを目標）
- 全AI回答に「これは一般的な情報です。個別の判断は専門家にご相談ください」のディスクレーマーを常時併記

### 2-3. エスカレフォーム設計

- 入力済みの質問内容を初期値として引き継ぎ
- ユーザーは連絡先（メール）を任意入力
- 送信→管理画面で該当士業（弁護士／行政書士／社労士）が確認・対応
- 収集しない: 在留カード番号・画像、パスポート画像、その他の要配慮個人情報（MVPでは扱わない）

### 2-4. Bot と Web の使い分け

- **Bot（Facebook Messenger）**: 短い一般質問、定型FAQ、リンク誘導
- **Web（PWA）**: 詳細記事、エスカレフォーム、夫婦間トラブル、設定、認証
- Botでの会話が一定回数を超えたら「**詳細はこちら**」とWebリンクに誘導

### 2-5. 料金設計（フリーミアム）

| 項目 | 設定 |
|---|---|
| 基本利用 | **無料** |
| 無料枠 | **月3回まで** AIチャット送信可能 |
| 有料プラン | 月4回以上送信したい場合に有料（月額は別途決定。想定 980円/月） |
| 期間単位 | 暦月（毎月1日にカウンタリセット） |
| カウント対象 | ユーザーからAIに送信されたメッセージ数（Bot・Web 合算、同一 Facebook アカウント内で集計） |
| カウント対象外 | 飲食店カタログ閲覧、記事閲覧、エスカレフォーム送信 |
| **MVP扱い** | **フリーミアムは封印して全機能無料で開放**。料金体系・Stripe課金フローはPhase 2で有効化 |

**MVPでの実装方針**:
- データモデルとして月次カウンタ（`monthly_chat_count` 等）と Stripe 関連カラムは**残す**（既存実装活用）
- ただしミドルウェアやAIエンドポイントでの**送信制限ロジックは無効化**（常に通す）
- Phase 2 で送信制限の有効化＋Stripe課金フローの再開

## 3. プラットフォーム・アーキテクチャ

### 3-1. 形態

- **Web (PWA)**: メイン。Next.js 16 + React 19（既存実装活用）
- **Facebook Messenger Bot**: 入口・簡易質問対応
- **LINEは外す**（フィリピン人主流SNSはFacebook）

### 3-2. 技術スタック

| 層 | 採用技術 |
|---|---|
| フロントエンド | Next.js 16 + React 19 + Tailwind CSS + shadcn/ui |
| 多言語化 | next-intl（既存実装活用）。ja/en/tl の3言語 |
| バックエンド | Next.js API Routes |
| DB | Supabase (PostgreSQL + pgvector + Auth + RLS) |
| 認証 | Supabase Auth + Facebook OAuth |
| AI（生成） | **OpenAI GPT-4o-mini** |
| AI（分類器） | OpenAI GPT-4o or GPT-4o-mini（要ベンチ。リスク判定の精度次第で決定） |
| Embedding | OpenAI text-embedding-3-small（既存のVoyageから乗り換え） |
| Bot | Facebook Messenger Platform |
| 決済 | Stripe（フリーミアム月額課金、ただしMVPは封印） |
| 監視 | Vercel Analytics + Sentry |
| ホスティング | Vercel + Supabase |

### 3-3. デフォルト言語

- **日本語デフォルト**
- ヘッダーまたは初回画面で **英語・タガログ** に切替可能（永続保存）
- ブラウザ言語自動判別は参考、ユーザー選択を優先

## 4. 法務・士業連携

### 4-1. 提携士業

| 士業 | 主な担当領域 | MVP配置 |
|---|---|---|
| 弁護士 | 民事系（夫婦間、いじめ、損害賠償）／利用規約・プラポリ監修 | メイン |
| 行政書士 | VISA・在留資格・帰化・行政手続き全般 | メイン |
| 社労士 | 社会保険関連 | サブ（個別質問エスカレ用） |
| 税理士・その他 | 必要に応じネットワーク経由で追加 | Phase 2 |

### 4-2. 責任分界

- ノヴァリスは「**情報プラットフォーム提供者**」のポジションを明確化
- 士業との契約に「士業の判断・対応はノヴァリスの責任ではない」条項を入れる
- 各士業は自身の業界保険（弁護士賠償責任保険等）に加入していることを契約条件に

### 4-3. 利用規約・プライバシーポリシー

- 弁護士監修で標準的なものを 3言語（日/英/タガログ）で作成
- 「AI回答は一般的情報であり法的助言ではない」を明記
- 起動時にチェックボックス1回で同意（永続保存）。重い同意フローにはしない

## 5. データ・セキュリティ方針

### 5-1. データ分類

| 分類 | 内容 | MVP扱い |
|---|---|---|
| 収集する（必須） | **Facebook ID**、表示名、メールアドレス | ○ |
| 収集する（運用で生成） | AIとの会話履歴、月次チャットカウンタ、エスカレフォーム入力、Stripe ID（Phase 2有効化時） | ○ |
| 収集しない | 在留カード番号・画像、パスポート、犯罪被害情報の生データ | × |

### 5-2. アカウント方針（1人1アカウント）

- **Facebook OAuth ログイン必須**（既存実装を継続）
- `profiles.facebook_id` に **UNIQUE 制約**を維持し、1人1アカウントを担保
- 同一 Facebook アカウントでの複数デバイス利用は**許容**（デバイス制限はかけない）
- **既存のデバイスフィンガープリント機能（`device_sessions` テーブル、`/api/auth/device-check` 等）は撤去**（永住者の家族共用と相性が悪く、過剰）

### 5-3. セキュリティ標準

- TLS 1.3
- Supabase RLS によるテナント分離
- 標準的なAES暗号化（要配慮個人情報を扱う場合のみフィールドレベル）
- アクセスログ記録
- 賠償責任保険・PCI DSS対応はMVP対象外（必要時に検討）

## 6. 段階リリース計画

### Phase 1（MVP / 想定 2〜3ヶ月）
- 上記7機能のローンチ
- 提携士業: 弁護士＋行政書士＋社労士（サブ）
- 言語: 日本語＋英語＋タガログ

- 社労士・税理士の本格運用、機能2の個別対応強化

- 不動産紹介（広告掲載型）
- 通訳予約・書類翻訳

### Phase 3 以降
- ECプラットフォーム（非食品から）
- 食品EC
- 育成就労制度（2027年施行）対応の更新
