# 弁護士監修依頼事項 (Novalis Phase 2 着手前)

_作成: 2026-05-27 / 監修依頼前最終版_

本ドキュメントは Novalis (在日フィリピン人向け生活支援アプリ) の Phase 2 着手に先立ち、業法・利用規約・AI 応答の境界について弁護士監修を依頼するための質問リストである。各項目は経営判断に直結するため、Yes/No だけでなく根拠条文と運用上の境界線を併せて提示いただきたい。

---

## 1. AI 応答と業法の境界

### 1-1 弁護士法 72 条 (非弁活動禁止) との関係

現状、AI は在留資格・離婚・相続・労働紛争などのテーマに対し「一般的な情報」を返している。具体的な事実関係に基づく法的判断や書類作成代行は escalation 文言で士業誘導しているが、その手前のグレーゾーンが論点。

質問。一般的な法律情報の提示 (例:「在留資格更新は期限 3 ヶ月前から申請可能」) は非弁にあたらないという理解で正しいか。あたるとすれば、どこから具体的法律事件性が発生し、それを AI 側で検知すべきラインはどこか。

### 1-2 行政書士法 1 条の 2 / 1 条の 3 との関係

入管手続き、車庫証明、各種許認可申請の案内が論点。

質問。在留資格申請の手順説明、必要書類リスト提示、申請窓口案内は行政書士法に抵触するか。代行ではなく情報提供に留めれば問題ないという解釈は成立するか。

### 1-3 社労士法 27 条 / 税理士法 52 条との関係

健康保険、年金、雇用保険、確定申告、年末調整、源泉徴収などのテーマで AI が応答中。

質問。社会保険手続きの一般的説明、税務上の論点整理 (具体額の計算なし) はそれぞれの業法独占業務に該当するか。具体的金額計算や申告書作成代行は明確に独占業務側だが、その手前の情報提供までの範囲を明確化したい。

### 1-4 不動産業との関係 (Phase 2 で予定)

Phase 2 で広告掲載型の不動産紹介 (外国人入居可能物件) を予定。

質問。広告掲載のみで取引仲介はしない場合、宅建業法上の免許は不要という理解で正しいか。提携不動産会社からの送客手数料を受け取る場合の規制は。

## 2. Escalation 設計の妥当性

### 2-1 Escalation 判定の二条件

現状の運用ポリシー (memory: project_filipino_chat_conversational_policy 参照)。Escalation は「具体的事実 + 専門家判断必須」が両方揃ったときのみ最終手段として発動。それ以外は AI が会話を続行する default。

質問。この二条件の組み合わせで escalation を抑制する設計は、業法上「相談に応じるべき具体事案を放置している」と評価されるリスクはあるか。逆に escalation を増やしすぎると士業への斡旋業に近づく懸念は。

### 2-2 Escalation 文言の最終承認

現行の暫定 escalation 文言を `docs/handoff.md` および本番アプリから参照可能。

質問。当該文言は法的に妥当か。修正案があれば具体的に。特に「専門家への相談を推奨します」というニュアンスが斡旋に近づかないかを確認したい。

### 2-3 Phase 2 で予定する Escalation スコアリング

現状は単一メッセージで判定するが、Phase 2 では直近 N ターンの累積スコアで判定する案あり。閾値を下げると escalation が増え、上げると AI 応答継続が増える。

質問。累積判定により「同一案件について複数ターン AI が応答した後で escalation」となるケースが増える。これは「AI が一度法的判断的な応答をしてから士業に振った」と評価されると、AI 応答部分が問題視される可能性はあるか。

## 3. 専門家紹介の取り扱い

### 3-1 斡旋業 / 紹介業の該当性

experts テーブルに登録された弁護士・行政書士・社労士・税理士を、ユーザの相談内容に応じて表示している。現状は無償マッチング、Phase 2 で予約 / 紹介手数料モデルを検討中。

質問。無償マッチングであっても「業として」反復継続的に行う場合、何らかの規制対象になるか。有償化 (紹介手数料受領) する場合は具体的にどの法令を確認すべきか (弁護士は非弁提携の問題があるはず)。

### 3-2 弁護士紹介の固有規制

質問。弁護士法 72 条との関係で、Novalis から弁護士への有償紹介は提携形式 (非弁提携) として NG という理解で正しいか。それを回避する運営モデルとして、ユーザから定額利用料を取り、その対価として弁護士情報提供を行うモデルは可能か。

## 4. Disclaimer / 免責の妥当性

### 4-1 AI 応答末尾の disclaimer

現行: 「これは一般的な情報であり、個別の判断は専門家にご相談ください。」(i18n 全言語版あり、`apps/v2/src/messages/{ja,en,tl}.json` の chat.disclaimer に格納)

質問。当該文言で AI 応答に基づく誤判断による損害について Novalis の責任を免責できるか。修正案があれば。特にフィリピン人ユーザが第一言語ではない日本語で理解する場合の説明十分性を懸念。

### 4-2 利用規約での免責条項

apps/v2/src/app/[locale]/legal/terms に暫定文言を掲載。

質問。AI 応答の正確性、専門家マッチング結果、第三者サービス (Google Form 問い合わせ、Komoju 課金) の障害について、現行の免責条項で十分か。

## 5. 利用規約 / プライバシーポリシー / 規約まわり

### 5-1 全文の法的妥当性

apps/v2/src/app/[locale]/legal/{terms,privacy} の暫定文言を全文監修依頼。日本語版がマスターで、英語 / Tagalog 版は翻訳。

質問。日本法準拠で在日フィリピン人ユーザを対象とする本サービスにおいて、現行の利用規約 / プラポリは最低限の法的保護として十分か。修正必要箇所と優先度を明示いただきたい。

### 5-2 同意取得フロー

新規登録時に「利用規約・プライバシーポリシー・18 歳以上」の 3 つに同意。`consent_logs` テーブルに同意 IP / UA / timestamp を記録。anonymous user は同意フロー skip 設計 (テストモード扱い、age_verified 自動 true)。

質問。同意取得方法 (チェックボックス 3 つ、それぞれ同意必須) は適法か。anonymous user の同意 skip は、anonymous user が anonymous であることを画面上明示している (テストモードバナー) ことで問題ないか。

### 5-3 18 歳未満排除の運用

age_verified を申告制で取得。実年齢確認 (本人確認書類) は未実装。

質問。18 歳未満排除の根拠を申告制のみとする運用は、Novalis の取り扱う情報領域 (在留資格、税務、労務など) において妥当か。本人確認書類取得を必須化すべき閾値があるか。

## 6. 個人情報保護法 (改正 PIPA) 適合性

### 6-1 PII 検出 / ブロックの実装

ユーザがチャットに個人情報 (在留カード番号、パスポート番号、電話番号、メールアドレス) を送信した場合、PII detector で検出し送信ブロック。検出ロジックは `apps/v2/src/lib/ai/pii-detect.ts`。

質問。当該実装は個人情報取り扱いの観点から十分か。Phase 2 で「マスクして処理続行」オプションを追加する案があるが、許される範囲はどこまでか。

### 6-2 海外データ移転の取り扱い

Google Gemini API (US / EU リージョン)、Supabase (ap-northeast-1 = 東京)、Vercel (US) を使用。LLM への入力プロンプトは Google サーバへ送信される。

質問。チャット内容を含む個人データの Google への送信について、ユーザ同意取得 / 通知義務として何が必要か。プライバシーポリシーへの記載で要件を満たすか、別途明示同意が必要か。

### 6-3 データ保持 / 削除

profiles, conversations, messages, consent_logs に保持。anonymous user は将来一括削除予定 (`DELETE FROM auth.users WHERE is_anonymous=true`)。permanent user のデータ削除リクエスト対応フローは未実装。

質問。GDPR / 改正 PIPA 上の削除権 (right to erasure / 利用停止請求) に対応するため、Phase 2 で何を実装すべきか。最低限の対応で済む線 (手動対応 + 規約への明記) はどこか。

## 7. その他

### 7-1 業法に抵触しない範囲での「最大限のサービス提供」の指針

Novalis の戦略は AI による情報提供で 80% を解決し、残り 20% を士業へ送るモデル。

質問。Novalis のような AI 主体サービスが業法上のグレーゾーンで提供範囲を最大化するには、運用上どのような自衛策が必要か (具体的事例での免責文言、escalation 強制発動の閾値、ログ保存範囲など)。

### 7-2 監修費用 / 監修後のアップデート対応

質問。本リスト全項目の監修にかかる費用見積もりは。Phase 2 リリース後、AI 応答パターンや escalation ロジック変更時の継続監修体制をどう組むのが現実的か (顧問契約 vs スポット契約)。
